# Status Page Setup

Target public URL:

`https://status.oncokind.com`

## Recommended monitors

- Main site: `https://www.oncokind.com`
- Authentication flow
- Report processing / API health

## Suggested setup

1. Create the status page with Instatus or Upptime.
2. Configure the public domain:

   `status.oncokind.com`

3. Add checks for:
   - Homepage uptime
   - Authentication health
   - Report processing or API endpoint health
4. Confirm the public page is reachable before launch.

The app footer already links to `https://status.oncokind.com`, so the external status page should be provisioned before release.
