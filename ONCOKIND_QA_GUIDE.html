Security & Data Isolation

RLS verification · API auth checks
RLS Isolation — Run in Supabase SQL Editor
-- Confirm each user only has their own cases (no cross-user leakage) SELECT p.email, COUNT(c.id) as case_count FROM public.profiles p LEFT JOIN public.prior_auth_cases c ON c.user_id = p.id WHERE p.email LIKE 'qa-%@oncokind.com' GROUP BY p.email ORDER BY p.email; -- Expected: only qa-professional and qa-admin rows have case_count > 0
API Auth Check — Run in Browser Console (logged out)
fetch('/api/prior-auth/cases').then(r => r.json()).then(console.log) // Expected: { error: "Unauthorized" }
